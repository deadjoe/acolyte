# 综合偏见、误导性、隐藏意图检测框架 v4

此框架旨在指导AI对中文文章进行系统化分析，评估可能存在的偏见、误导性内容及隐藏意图。请严格按照结构分析并输出Markdown格式结果。

## 分析前准备

1. **文章类型**：确定文章属于以下哪种类型
   - 新闻报道
   - 评论文章
   - 学术论文
   - 广告/营销
   - 社交媒体内容
   - 其他类型（请具体说明）

2. **目标受众**：
   - 分析文章使用的专业术语、语气和背景知识假设
   - 根据这些特征推断目标受众（如普通公众、专业人士、特定群体等）

3. **背景与时间**：
   - 文章发布时间（如已知）
   - 相关事件时间线（若文中提及）
   - 是否属于特定讨论、争议或事件系列

4. **文章类型调整指南**：
   - **新闻报道**：重点关注A.2(视角多样性、消息来源)和B.1(事实准确性)
   - **评论文章**：接受适度观点表达，重点评估A.3(框架效应)和C.3(隐形框架)
   - **学术论文**：严格评估B.1(事实准确性)和B.2(逻辑谬误)，对引用要求最高
   - **广告/营销**：预期存在正面倾向，重点关注B.3(混淆技巧)和C.2(情感操纵)
   - **社交媒体**：考虑平台特性，对情绪化表达给予适当宽容

## A. 偏见检测（操作性分析）

### A.1 语言层面分析
1. **情绪化词汇检测**：
   - 标记所有带有明显情感色彩的形容词和副词（如"可怕的"、"荒谬的"）
   - 计算正面/负面词汇比例，确定是否明显倾向
   - 识别绝对化词汇（"总是"、"从不"、"全部"、"没有一个"等）并计数

2. **称谓与标签分析**：
   - 识别对人物/群体的称谓方式是否一致或存在差异
   - 检查是否使用带有贬义的标签或污名化词汇
   - 分析是否使用不对称的描述（如一方用专业头衔，另一方用简称）

3. **语言变体影响评估**：
   ```
   示例：称某国人为"难民"vs"移民"vs"新居民"传达不同印象
   ```
   - 检查是否选择特定词汇框架讨论对象
   - 分析替代词汇可能传达的不同印象

### A.2 叙事结构分析
1. **视角多样性量化**：
   - 统计支持不同立场的段落比例
   - 评估反对观点是否被完整呈现（还是仅作简化概括）
   - 检查是否给予不同观点相近的论述深度和空间

2. **消息来源评估**：
   - 识别并列出所有被引用的消息来源
   - 评估消息来源是否多元（机构类型、立场多样性）
   - 检查是否存在未具名/模糊来源（"有人说"、"据了解"、"专家认为"）

3. **数据使用分析**：
   - 检查是否只选择性呈现支持特定观点的数据
   - 评估数据是否有完整背景（时间范围、样本大小、数据来源）
   - 分析数据表述是否精确（绝对数vs相对数、百分比等）

### A.3 框架效应分析
1. **标题与内容一致性**：
   - 评估标题是否准确反映正文内容或过度强调某一方面
   - 检查副标题/引言是否设置了偏向性预期

2. **叙事顺序效应**：
   - 分析文章开头是否建立了特定框架
   - 评估支持/反对观点的呈现顺序是否影响读者印象
   - 检查结论部分是否平衡或偏向特定立场

3. **案例选择评估**：
   - 分析所选案例是否具有代表性或为极端个例
   - 检查是否使用情感化案例代替系统性数据
   - 评估是否呈现了反面案例

> **执行指南**：对每个发现，引用原文关键句（用引号标注），解释其如何构成偏见，并评估严重程度（1-5分，5分最严重）。这些局部评分用于具体问题描述，不直接转换为最终的0-10分量化评分。请注意参考前文确定的文章类型，对不同类型文章应用适当宽严标准。

## B. 误导性内容检测

### B.1 事实准确性分析
1. **事实陈述识别与分类**：
   - 区分文中的观点陈述vs事实陈述
   - 将事实陈述分类为：
     a) 可直接从文内验证
     b) 需外部信息验证
     c) 模糊或混合了事实与观点

2. **上下文完整性检查**：
   - 检查数据是否有完整背景信息（时间、地点、条件、来源）
   - 评估引用是否完整或存在选择性截取
   - 分析是否省略了理解数据所必需的比较基准
   ```
   示例误导：声称"X增长了200%"但未提供基数，原基数可能极小
   ```

3. **混合内容识别**：
   - 检查是否在不明确区分的情况下混合了：
     a) 已确认事实与推测/预测
     b) 观点与事实
     c) 主流共识与边缘观点
   - 识别使用模糊修饰语掩盖不确定性的表述（"可能"、"据信"）

### B.2 逻辑谬误分析
1. **常见谬误识别**：针对以下模式进行匹配并标记：
   - **因果谬误**：将相关混淆为因果、忽略第三变量
     ```
     示例："自从A发生后，B也发生了，所以A导致了B"
     ```
   - **以偏概全**：从个例快速概括到整体
     ```
     示例："一个X做了Y，所以X群体都会做Y"
     ```
   - **稻草人谬误**：简化或歪曲对方论点再进行反驳
     ```
     示例："支持政策X的人认为[极端简化版本]，但这显然是错误的"
     ```
   - **虚假二分法**：将复杂问题简化为两个极端选项
     ```
     示例："要么支持方案A，要么就是反对进步"
     ```
   - **滑坡谬误**：假设一小步将不可避免地导致极端后果
     ```
     示例："如果允许X，最终将导致Y（一系列未证明必然性的极端后果）"
     ```

2. **复杂问题简化检测**：
   - 评估是否过度简化多因素议题
   - 检查是否忽视已知的关键反驳证据
   - 分析是否使用简单解决方案应对复杂问题

### B.3 混淆技巧识别
1. **模糊语言分析**：
   - 标记使用模糊定义或可多重解释的关键词
   - 识别数量模糊表述（"许多人"、"大量证据"等）
   - 检测无法验证的宣称（"最好的"、"领先的"等）

2. **转移与分散技巧**：
   - 识别议题转移模式（从主要问题转到次要或无关问题）
   - 检测情绪分散（使用情感激发转移理性分析）
   - 分析是否使用假设性问题回避直接指控

3. **伪专业性评估**：
   - 检查是否使用不必要的专业术语或行话
   - 评估引用的"专家"是否具有相关专业背景
   - 分析是否使用科学化语言掩盖缺乏实质内容

> **执行指南**：为每个发现标注确信度（高/中/低），提供原文引用证据，并解释为何构成误导。评估严重程度（1-5分，5分最严重）。注意：如果无法验证某项事实，明确标记为【需外部验证】，不要试图猜测或推断真伪。请根据文章类型调整标准，对学术论文的事实准确性要求最高。

## C. 隐藏意图检测

### C.1 信息选择性呈现分析
1. **重要背景省略检测**：
   - 识别理解议题所需但被省略的关键背景
   - 评估历史背景/前情是否完整呈现
   - 检查是否省略了与主论点矛盾的重要事实
   ```
   示例：批评政策效果但不提及实施时间太短无法合理评估
   ```

2. **统计与数据选择检查**：
   - 评估所选时间段/样本是否有代表性
   - 检查数据呈现方式是否可能误导（图表比例、起始点选择等）
   - 分析是否使用不常规统计方法无合理解释

3. **证据平衡评估**：
   - 检查正反证据是否均衡呈现
   - 评估是否对支持观点的弱证据给予过高权重
   - 分析是否对反对观点的强证据给予过低权重

### C.2 情感操纵分析
> **注意**：需区分作者正常情感表达（如哀叹、愤慨）与刻意操纵（如无充分证据却反复渲染恐慌）。如无法确定其主观意图，请标记为低确信度或"有待进一步信息"。

1. **恐惧/焦虑诱导识别**：
   - 检测是否使用威胁或后果警告
   - 评估是否强调不确定性或风险而缺乏比例感
   - 分析是否使用"最后机会"/"危急时刻"等紧迫性框架

2. **身份与价值操纵检测**：
   - 识别对特定群体身份认同的暗示性诉求
   - 检查是否使用"我们vs他们"的二元对立框架
   - 评估是否暗示采纳特定观点是某种价值观/身份的"必然选择"

3. **情感替代分析**：
   - 检测是否使用强烈情感叙述替代事实或逻辑
   - 评估是否利用读者现有情感/偏见
   - 分析是否使用煽情案例代替系统性证据

### C.3 隐形框架分析
1. **未声明假设识别**：
   - 检测文章基于哪些未经证明的假设
   - 评估这些假设对论证的关键程度
   - 分析这些假设是否具有争议性
   ```
   示例："我们需要提高X"（隐含假设：当前X水平不足）
   ```

2. **比喻与类比评估**：
   - 识别文中使用的主要比喻/类比
   - 分析这些比喻如何框架问题/引导理解
   - 评估比喻的适当性与局限性

3. **价值加载语言检测**：
   - 识别包含内置价值判断的词汇使用
   - 检查是否使用道德暗示语言("应该"、"负责任的人会")
   - 分析是否将特定立场与普遍认可的价值观绑定

> **执行指南**：为每个发现提供1-5分的确信度评分并解释理由。引用原文支持分析，并说明可能的意图模式。注意避免过度解读或无证据的猜测。请考虑文章类型，如评论文章表达观点较为正常，而新闻报道则应更加中立。

## D. 量化评分系统

### 重叠问题处理原则
- 若同一文本证据在多个维度符合不同问题特征（如既有偏见又有误导），仅在最主要或影响最严重的一个类别中计入量化评分
- 在发现描述中可以交叉引用相关维度，但在最终量化评分时避免重复计入
- 评分说明中应注明:"为避免重复计算，同一问题证据仅计入最相关维度的评分"

### 评分样例示范
以下示例展示如何从局部1-5分严重度评分，转化为整体0-10分评估：

```
例如，在语言偏见分析中发现：
1. 使用贬低性称谓（严重度3分）
2. 大量情绪化词汇（严重度4分）
3. 极端对比框架（严重度5分）

综合考虑这些发现以及它们在文章中的普遍性，可能给语言偏见维度评估为7分（0-10分制），
而非简单取平均或相加。若这些问题仅出现在文章特定部分而非全文，可能评为5-6分。
```

### 基础指标评分（各项0-10分，0=无问题，10=问题明显）

#### 1. 偏见指数 (BI)
- 语言偏见 (0-10): ___ × 权重 0.35 = ___
- 叙事偏见 (0-10): ___ × 权重 0.35 = ___
- 框架偏见 (0-10): ___ × 权重 0.30 = ___
- **加权BI** = ___ （和）

#### 2. 误导性指数 (MI)
- 事实准确性问题 (0-10): ___ × 权重 0.40 = ___
- 逻辑谬误 (0-10): ___ × 权重 0.35 = ___
- 混淆技巧 (0-10): ___ × 权重 0.25 = ___
- **加权MI** = ___ （和）

#### 3. 隐藏意图指数 (HI)
- 信息选择性呈现 (0-10): ___ × 权重 0.35 = ___
- 情感操纵 (0-10): ___ × 权重 0.35 = ___
- 隐形框架 (0-10): ___ × 权重 0.30 = ___
- **加权HI** = ___ （和）

#### 4. 综合可信度分数 (CS)
- CS = 100 - [(BI + MI + HI) × 10 / 3]
- **最终CS** = ___

### 评分指南说明
- 局部分析中使用的1-5分评分用于标注具体问题严重程度
- 最终量化时的0-10分评分是对整个维度的综合评估，需考虑问题的严重性、普遍性及文章类型
- 较低分数(0-3)表示问题轻微或几乎不存在
- 中等分数(4-7)表示存在明显但非极端的问题
- 较高分数(8-10)表示问题严重且普遍存在

### 可信度分类标准
- **90-100**：高度可信（极少偏见、无明显误导、意图透明）
  ```
  特征：多元观点、完整引用、平衡呈现、适当限定、数据完整
  ```
- **70-89**：较为可信（轻微偏见或误导，整体仍可靠）
  ```
  特征：略偏视角但呈现多方观点、个别数据缺乏完整背景、少量情绪化表达
  ```
- **50-69**：中等可信（存在明显偏见与误导，需谨慎对待）
  ```
  特征：观点不平衡、部分事实断章取义、存在明显逻辑谬误、情绪化论述
  ```
- **30-49**：可信度较低（明显偏见，多处误导，存在可辨识意图）
  ```
  特征：单一立场、多处事实问题或曲解、频繁使用情感诉求替代论证
  ```
- **0-29**：可信度不足（显著问题，需要高度批判性阅读）
  ```
  特征：高度宣传性质、大量情绪煽动、多处误导性陈述、完全不平衡
  ```

## E. 分析输出格式

### 1. 分析前背景总结
简要描述文章背景、类型、目标受众（100字以内）

### 2. 偏见检测发现
列出3-5个最显著偏见问题，每个包含：
- 具体引文（引号标记）
- 问题解释
- 严重程度评分(1-5)
如未发现明显问题，则说明"未发现明显偏见问题"

### 3. 误导性内容检测
列出3-5个最显著误导问题，每个包含：
- 原文引用
- 误导性解释
- 确信度（高/中/低）
- 严重程度评分(1-5)
如未发现明显问题，则说明"未发现明显误导内容"

### 4. 隐藏意图检测
列出3-5个最显著隐藏意图线索，每个包含：
- 相关证据（引文或模式）
- 可能意图分析
- 确信度评分(1-5)及理由
如未发现明显问题，则说明"未发现明显隐藏意图"

### 5. 整体评估
200字以内总结，包括：
- 文章总体可信度评价
- 主要问题概述
- 读者应如何看待此内容的建议
- 改进建议（若适用）

### 6. 量化评分
提供必要的计算过程，然后必须在最后以下面的JSON代码块格式输出量化评分结果（此格式对自动化处理至关重要）：

```json
{
  "偏见指数": {
    "名称": "BI",
    "分数": [分数值]
  },
  "误导性指数": {
    "名称": "MI",
    "分数": [分数值]
  },
  "隐藏意图指数": {
    "名称": "HI",
    "分数": [分数值]
  },
  "综合可信度": {
    "名称": "CS",
    "分数": [分数值],
    "分类": "[可信度分类]"
  }
}
```
注意：
请用实际计算的数值替换[分数值]和[可信度分类]
确保JSON格式严格正确，不要添加任何额外文本

### 7. 分析局限与不确定性
标明：
- 无法确定的关键点
- 需要额外信息验证的方面
- 分析的主要局限性
- 特别注明需要外部事实核验的具体内容（加粗标记【需外部验证】）

## F. 自校验清单

分析完成后，请检查以下方面并调整（无需在输出中包含此清单）：

1. **平衡性检查**：
   - 我是否对所有观点应用了相同标准？
   - 是否考虑了文章类型的语境适当性？
   - 是否过度强调了小问题或忽视了大问题？

2. **证据充分性**：
   - 每个判断是否都有文本证据支持？
   - 是否存在我解读过度的地方？
   - 我的确信度评级是否与证据强度匹配？

3. **分析盲点**：
   - 我可能受模型训练数据偏见影响的领域？
   - 我可能缺乏文化/专业背景导致误解的地方？
   - 是否有我忽视的合理替代解释？

4. **避免重复计算**：
   - 是否有同一问题被多次计入不同维度？
   - 在最终量化时是否仅将每个问题计入最相关的一个维度？

*此框架专为LLM辅助内容分析设计，旨在提供系统化、结构化的评估。若有不确定之处，请明确标示"不确定"或"信息不足"，避免推测。分析应基于文本内容而非个人价值立场。遇到无法验证的事实陈述，请明确标记为【需外部验证】而非尝试判断真伪。*
